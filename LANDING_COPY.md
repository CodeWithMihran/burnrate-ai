This copy is written to support that exact product shape.

## Feature Cards

### 1. Structured spend review

Every tool gets its own evaluation space so teams can compare plan fit, current spend, and likely next steps without digging through scattered subscriptions.

### 2. No-noise recommendations

Recommendations are tied to plans, seats, and spend context so the output feels more like an operator memo than a generic AI answer.

### 3. Shareable audit results

Each finished audit can be saved and shared publicly, turning the result into a lightweight decision artifact for teammates or leadership.

## Problem Section Headline

AI tools are everywhere. Spend discipline usually is not.

## Problem Section Body

Startup teams adopt AI tools quickly because the upside is obvious. The downside appears later: overlapping subscriptions, mismatched seat counts, and retail pricing paid across too many tools at once. BurnRate.ai is meant to clean that up.

## FAQ

### Do I need to sign up before using it?

No. The audit is available before any report-capture or follow-up request.

### What tools does it support?

The current version supports ChatGPT, Claude, Cursor, GitHub Copilot, Gemini, Windsurf, and API-direct workflows for OpenAI and Anthropic.

### Is the recommendation engine AI-driven?

No. The recommendation logic is rule-based. AI is only used for the short narrative summary layer.

### What if my stack is already well-optimized?

Then the product should say so. BurnRate.ai is designed to avoid inventing fake savings just to make the output feel exciting.

### What happens after the result?

You can review the recommendations, copy the public share URL, save the report, or follow a stronger next-step path if the result surfaces meaningful savings.
